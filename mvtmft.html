Memory Allocation Techniques

Multiprogramming with Fixed Tasks (MFT)

MFT divides memory into fixed-size partitions. Each process is allocated to a partition that fits its size. The key characteristic of MFT is that partitions are created at system startup and remain fixed in size and location.

Advantages

  • Simple to implement
  • Low overhead for memory management
  • Predictable performance
  • No external fragmentation

Disadvantages

  • Internal fragmentation (unused memory within partitions)
  • Limited flexibility in process sizes
  • Poor memory utilization if process sizes vary greatly
  • Maximum process size limited by largest partition

MFT Example

Memory: 1000KB, Partition size: 250KB → 4 partitions

Processes:

  • P1: 200KB → Fits in partition (50KB internal fragmentation)
  • P2: 300KB → Too large, cannot be allocated
  • P3: 100KB → Fits in partition (150KB internal fragmentation)

Multiprogramming with Variable Tasks (MVT)

MVT allocates memory dynamically to processes as needed. Memory is treated as one large pool of free memory, and processes are allocated exactly what they need. The key characteristic of MVT is that memory is allocated and freed as processes start and terminate.

Advantages

  • No internal fragmentation
  • Better memory utilization
  • Can accommodate processes of any size (up to total memory)
  • More flexible than MFT

Disadvantages

  • External fragmentation (small unusable blocks of free memory)
  • More complex memory management
  • Requires compaction to reduce fragmentation
  • Performance overhead for allocation/deallocation

MVT Example

Memory: 1000KB

Processes:

  • P1: 200KB allocated → 800KB free
  • P2: 300KB allocated → 500KB free
  • P1 terminates → 200KB + 500KB free (non-contiguous)
  • P3: 400KB cannot be allocated (external fragmentation)
